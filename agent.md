# Agent Plan

## Project Goal

Build a local static learning site for entering Chinese metaphysics through classic texts, with the broader goal of training a higher-level framework for understanding reality.

This is not just a book-notes site. The books provide the source structure, while the site helps the learner move from original text, to plain-language understanding, to logical models, to real-life observation.

## Core Direction

- Use the original book outlines as the primary navigation.
- Do not invent top-level modules first. Extract models from chapters and aggregate them secondarily.
- Treat metaphysics as cultural learning, systems thinking, and observation practice. Do not make prediction, fortune-telling, or divination claims.
- Keep modern explanations original. Do not copy full modern translations or commentaries.
- Current manual-content milestone is complete: all chapters now have manual plain-language translation and logic notes.
- Quality requirement for every manual note: plain-language and logic must be simple, concrete, and easy for a beginner to understand.
- Special priority: `周易` is hard enough that it must be rebuilt around a beginner foundation layer before continuing normal chapter production.

## Three-Month Learning Path

1. Month 1: `道德经`
   - Goal: build a bottom-level worldview.
   - Focus: reduce inner friction, understand `顺势`, observe human nature and system patterns, move from forceful resistance to leverage.

2. Month 2: `孙子兵法`
   - Goal: understand real-world competition.
   - Focus: information asymmetry, game dynamics, rhythm, human nature, resources, timing.
   - Framing: read it as a reality and society operating manual, not only as a military text.

3. Month 3: `周易`
   - Goal: learn change and cycle thinking.
   - Focus: yin-yang, changing states, timing, position, reversal, risk boundaries.
   - Framing: start from change thinking, not fortune-telling or hexagram casting.

## Zhouyi Rebuild Direction

The user clarified that `周易` cannot be treated like ordinary chapter notes. It needs a foundation layer first, then chapter-level close reading.

Main principle:

- First build a clear physical worldview: what `卦`, `爻`, `阴阳`, `位`, `时`, `变`, `中`, `正`, `应`, `承`, `乘`, `吉凶悔吝` mean in concrete life.
- Then explain each hexagram and each line.
- The user should not have to infer the system from scattered line notes.

Main agent understanding:

- `周易` is a framework for reading changing situations. Its first useful layer is not prediction; it is a way to see force, position, timing, relation, and risk.
- A hexagram is not a mystical answer. It is a structured snapshot: lower/upper forces, six stages, and the direction in which a situation may move.
- A line is not an isolated saying. It only becomes clear when read inside the six-stage chain: beginning, emerging, pressure, testing, maturity, and excess.
- `九` and `六` should be explained as functional modes, not moral ranks. `九` is like sunlight, a seed pushing up, or an engine outputting force. `六` is like moonlight receiving, earth holding roots, or a road carrying traffic.
- `阴阳` should not be flattened into active/passive. More accurate: one side initiates and releases energy; the other receives, shapes, and lets it land.
- Do not teach `阳=主动、阴=被动` as the final formula. A better beginner formula is: `阳` is outward release and visible push; `阴` is receptive capacity and form-giving support. The moon is not "doing nothing" when it receives light, and the earth is not weak when it holds roots; without that receiving field, yang force cannot become a real outcome.
- Every `周易` chapter must first tell the reader what physical scene or real-world structure it resembles, then explain what each section says.
- The rebuild target is not more text. It is clearer text: first physical picture, then change chain, then one-to-one plain-language explanation, then logic model, reality example, and one-line summary.

Latest user emphasis:

- `周易` is difficult enough that readers need a concentrated foundation lesson before the 64 hexagrams.
- The foundation lesson must build a physical worldview: how forces arise, receive, accumulate, shift position, mature, and overrun.
- It must state the core viewpoint, core principles, and essence before any individual hexagram notes.
- Each hexagram then explains every source segment directly: what the words mean, where it sits in the change chain, and what concrete real-world situation it resembles.
- The priority is to rebuild all `周易` chapters around this foundation-first method, not merely fill missing translations.

### Zhouyi Foundation Layer

Before continuing large-scale `周易` chapter work, create a concentrated beginner section that explains:

- `易`: not fortune-telling first, but a way to observe change, timing, position, and risk.
- `卦`: a snapshot of a situation, like a weather map. It does not say one fixed result; it shows forces, positions, and likely movement.
- `爻`: one of six layers inside a hexagram, like six steps of a process. Read from bottom to top: hidden -> emerging -> pressured -> testing -> mature -> overdone.
- `九`: yang-line notation. Use physical examples: sun emitting light and heat, a seed pushing upward, an engine providing thrust. It means initiating, pushing, expanding, showing up.
- `六`: yin-line notation. Use physical examples: moon receiving and reflecting light, earth holding water and roots, a road carrying traffic. It means receiving, shaping, stabilizing, letting things land.
- `阴阳`: not simply active/passive, not male/female value ranking. It is two functions in one system: one side pushes energy out, the other side receives and forms it. A good system needs both.
- When explaining `九/六`, use physical scenes: the sun warms and starts growth, the moon receives and reflects light, the seed pushes upward, the soil holds roots and water, the engine outputs thrust, and the road carries traffic. These examples make clear that `六` is not merely passive; it is the field that lets force settle and take shape.
- `位`: where a force sits. Same ability at a low, middle, or high position has different meaning.
- `时`: whether the moment is ready. A right action at the wrong time becomes a mistake.
- `中`: not extreme, staying near the useful center.
- `正`: role and position match; what you do fits where you are.
- `应`: two positions can respond to each other, like sender and receiver.
- `承 / 乘`: lower and upper positions affect each other; support from below is different from pressure from above.
- `吉 / 凶 / 悔 / 吝 / 咎`: risk labels, not supernatural verdicts. They mean likely smoothness, danger, regret, stuckness, or later trouble.

This foundation must be written with concrete images:

- sun and moon
- land and water
- seed, root, branch, fruit
- weather building into storm
- traffic light and road capacity
- startup from idea to launch to overexpansion
- team role from junior contributor to lead to overreaching executive

Avoid abstract-only explanations like `阴阳是宇宙本体` or `卦象代表万物生成`. The site should teach a beginner to see a situation physically and structurally.

Core worldview to keep visible:

- A situation is not a static object; it is a moving field made of force, position, timing, relation, and boundary.
- `阳` is the side that emits, starts, presses, expands, or exposes. It is like sunlight warming the ground, a seed breaking soil, or an engine applying thrust.
- `阴` is the side that receives, holds, shapes, buffers, or lets force settle. It is like earth holding roots, water taking the shape of a container, or the road carrying traffic.
- `爻` is a stage in that field. A six-line hexagram is a vertical process map: hidden beginning, first emergence, pressure and responsibility, testing and transition, mature center, and overextension.
- `吉凶悔吝` are not magical verdicts. They are risk labels: smooth path, danger, regret after seeing the cost, or narrow/stuck condition.
- The essence of reading `周易`: first see the structure and motion, then judge whether the action fits its position and timing.

### Zhouyi Chapter Shape

Each `周易` hexagram should be reworked in this order:

1. `基础读法`: explain the hexagram image and core tension before showing line details.
2. `物理类比`: give one concrete scene, such as water waiting before crossing, thunder under earth, fire above lake, wind entering wood.
3. `变化链条`: summarize how the six lines move from bottom to top.
4. `逐段白话`: explain `卦辞`, `彖曰`, `大象曰`, and each `爻辞 + 小象曰` one by one.
5. `逻辑模型`: explain how the line fits the whole change chain.
6. `现实例子`: one concrete example for the line or paragraph.
7. `总结`: one short sentence to remember.

Chapter notes must not jump straight into line-by-line detail without first saying what world model the reader is entering.

## Book Foundation Layers

Every core book should now have a beginner-facing foundation layer before chapter reading. The foundation is not a decorative intro; it tells the reader which physical or real-world model to hold in mind while reading the original text.

### Dao De Jing Foundation Direction

`道德经` should be introduced as a low-friction worldview:

- `道`: the natural route or flow of a system, like water following slope or people following incentives.
- `名`: a temporary label, not the thing itself; always check behavior, conditions, and results behind a label.
- `无为`: not doing nothing, but removing disruptive over-control and acting only where action helps the flow.
- `水 / 谷 / 柔弱`: low position, receiving capacity, detour, and persistence can gather more power than direct collision.
- `反转`: when something becomes too full, too hard, or too extreme, it tends to produce the opposite movement.
- `欲望 / 内耗`: excess desire, face, comparison, and anxiety consume judgment before the external problem is solved.
- `德`: stable embodied force; a person or system repeatedly produces order, trust, and lower friction.

The reader should first ask: what is the flow, what is the blockage, what label might be misleading, what force is overfull, and where can a smaller action produce a larger effect.

### Sunzi Foundation Direction

`孙子兵法` should be introduced as a competition and decision manual, not as a love of conflict:

- `计`: compare conditions before moving; do not let emotion replace calculation.
- `五事七计`: turn the situation into checkable variables: alignment, timing, terrain, leadership, rules, strengths, weaknesses.
- `信息差`: reliable information reduces wasted action; distinguish signal from performance.
- `虚实`: resources are unevenly distributed; avoid hard points and find thin points.
- `势`: structure-created momentum; organize people, process, timing, and position so the situation rolls forward.
- `节奏`: fast, slow, direct, indirect, advance, and retreat all depend on timing and supply.
- `成本`: victory that burns cash, trust, attention, and team stamina may be a bad victory.
- `人性风险`: a leader's fear, anger, pride, rigidity, or softness can become the enemy's entry point.

The reader should first ask: what is being competed over, what is the real cost, who sees more clearly, where are the hard and soft points, and whether there is a lower-loss path than direct collision.

### Sunzi Historical Cases Direction

`孙子兵法` also needs historical cases, but they should not be mixed into chapter notes.

Placement:

- Put historical cases on a separate book-level page after `应用总结`.
- Route: `/sunzi/stories`.
- Left rail label: `历史案例 / 典故拆解`.
- Data source: `data/manual/book-pages/sunzi/stories.json`.
- Generated static JSON: `src/data/book-pages/sunzi/stories.json`.

Content standard:

- Do not quote long classical source text.
- Explain cases in plain modern Chinese.
- Every case should show: background, trigger, process, result, principle, execution problems, modern use, and review questions.
- The page should teach how `孙子兵法` works in messy real situations, not present stories as slogans.
- Important angle: real application usually fails because information is incomplete, timing is wrong, people resist, resources cannot keep up, or the winner cannot preserve results.
- Use historical cases to show the actual judgment chain: facts -> structure -> move -> risk -> result -> review.

Current first cases:

- `桂陵之战`: 围魏救赵, avoiding the hard point and hitting the pressure source.
- `马陵之战`: 减灶诱敌, information design plus opponent psychology.
- `官渡之战`: 烧乌巢, supply point as the real weak point.
- `赤壁之战`: 火攻不是点火, timing, alliance, wind, terrain, and execution conditions.

## Local Data Architecture

Use JSON as the long-term local source of truth for manual notes. Do not put hand-written translation or logic content into TypeScript files.

- `data/manual/chapters/**`: editable manual content. This is where `translater` and `logic` work.
- `src/data/content.ts`: lightweight book/chapter catalog for initial page render.
- `src/data/chapters/**`: generated chapter payloads for lazy loading. Do not edit by hand.
- `data/generated/content.json`: full generated payload for validation, export, and debugging.
- `data/generated/content-index.json`: lightweight generated index matching `src/data/content.ts`.

The app lazy-loads chapter bodies with Vite `import.meta.glob`. Initial render should load the catalog only; original text, manual translation, and logic notes load when the active chapter route is opened.

Durability rules:

- After any manual JSON edit, run `pnpm build:content`.
- Treat generated files as rebuildable output.
- Keep chapter-level JSON small and independent so CRUD stays cheap as the project grows.
- Preserve existing sibling fields in a manual chapter file. `translater` must not delete `logic`; `logic` must not delete `translation`.

Performance and maintainability rules:

- High performance: keep initial bundle focused on navigation and metadata; lazy-load chapter bodies.
- High availability: generated static JSON chunks must be valid after every build; page should show a readable loading/error state if a chapter chunk is missing.
- High extensibility: new books, new note fields, and later local CRUD screens should extend the JSON schema without forcing a full app rewrite.

## Agent Workflow For Long-Term Content

This project has many chapters: `道德经` 81 chapters, `孙子兵法` 13 articles, and `周易` 64 hexagrams. Content completion should be treated as a long-running editorial workflow, not a one-shot generation task.

Recommended sequence for each batch:

1. Main agent selects a small batch of chapters and confirms their `chapterId`.
2. `translater` writes or updates `data/manual/chapters/{bookId}/{chapterId}.json`.
3. `logic` writes or updates the same manual JSON file after translation exists.
4. Main agent runs `pnpm build:content` to regenerate lazy chapter payloads.
5. Main agent wires app code only if data shapes changed.
6. Main agent runs verification and checks the rendered page.

Batch size:

- `道德经`: 3-5 chapters.
- `孙子兵法`: 1 article.
- `周易`: 3-5 hexagrams.

Do not ask one agent to do another agent's job. If a chapter lacks translation, `logic` should wait. If a model/example is wrong, revise the `logic` field in the manual chapter JSON, not the `translation` field.

## Main Agent Operating Rules

The main agent owns coordination, progress tracking, review, and verification.

When the user says `继续`, continue the next unfinished batch immediately. Do not ask what to do next unless the local state is ambiguous or files are missing.

When the user says `一直做完`, keep working through the remaining chapters autonomously:

- Select the next batch.
- Assign `translater` first.
- After translation lands, assign `logic`.
- Review both outputs.
- Fix wiring or type issues.
- Run verification.
- Update progress notes.
- Continue to the next batch.

The main agent should not write translation or logic content itself unless a subagent is unavailable or the user explicitly asks. It should edit app/layout/data wiring, agent docs, scripts, and progress tracking.

Main agent review must reject notes that sound abstract, padded, or hard to understand. The required standard is: a beginner can read the white-language note, logic model, example, and summary without needing extra explanation.

## Content Unit

Each chapter, article, or hexagram should keep this structure:

- Original text
- Plain-language summary
- Logic notes
- Reality mappings
- Observation practice prompt
- Tags
- Source URL and license/source note

Manual additions should live under `translation` and `logic` in `data/manual/chapters/{bookId}/{chapterId}.json`. Generated fallback fields may remain in generated chapter payloads for chapters not manually completed yet.

## Important Constraints

- Keep the book outline as the main skeleton.
- Keep each book's foundation layer as a single `导读页`, not repeated inside every chapter. Routes: `/dao-de-jing`, `/sunzi`, and `/zhouyi` are book-level guide pages; chapter routes such as `/dao-de-jing/ddj-01` should focus on 原文、白话、关键词、逻辑模型、现实例子、总结。
- Keep model tags grounded in chapters.
- Preserve source attribution.
- Avoid unsupported modern copyrighted commentary.
- Avoid turning the project into an occult prediction tool.
- Prioritize readable Chinese UX over decorative landing-page design.
- White-language and logic content must be 通俗易懂: short sentences, concrete words, direct examples, and no empty transition phrases.
- Do not accept vague examples. Reality examples must make the model obvious.
- Do not use placeholder manual notes. If a chapter has not been read carefully, mark it pending instead of pretending it is complete.
- Current quality target is not more literary or more academic. The target is more direct, more visual, and easier to use: short sentences, common words, concrete analogies, fewer abstract nouns, fewer transition phrases, and more "this is like..." links to daily life.
